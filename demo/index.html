Embed remote apps without hand-rolled message glue.

`@crup/port` gives the host lifecycle control, origin checks, request-response semantics, and child-driven resize signals in a package small enough to stay out of the way.

    Live Demo

    One host, one child, one explicit protocol.

    The host page mounts an iframe, waits for `load`, sends `port:hello`, and only then opens the session for requests and events. The child accepts messages only after it sees a matching origin and instance id. 

    const port = createPort({
  url: childUrl,
  allowedOrigin: window.location.origin,
  target: '#demo-root',
  mode: 'inline'
});

await port.mount();
await port.call('system:ping');

    Protocol

    The envelope stays small. The guarantees stay strict.

    1

    `protocol` and `version` are validated before any routing happens.

    2

    Host messages are pinned to the mounted iframe window and the expected origin.

    3

    `instanceId` prevents sibling embeds from cross-handling each other.

    4

    Requests resolve through `replyTo` instead of ad hoc event naming.
